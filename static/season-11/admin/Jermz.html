Answer to part one: So currently there is no effective way to get 2 different committees to work together. Lets me put this from an internal view: Media finds out about events through mostly the media head announcing that we will have streams. At no point before that (unless a media person is also in events) does media know about the event. Similarly, an idea was given for integration with the website for our overlay. The person making the idea has no internal avenue to go to continue working on this idea and it is either dead right there or has to go externally to the DMs of a tech committee member. This is where I can come in. When an idea like that is thought of I want there to be a way to communicate with an available committee member to see if this idea is feasible and if it's within our scope to be accomplished. I first want to take the idea to the committee whose help is needed and find out if this idea is feasible. I then want to create an internal CSC avenue in Discord for these 2 parties to be able to talk further about the idea and, if both parties see this as a worthwhile venture, create a plan of action to get this idea into development. At that point, it will go on a Kanban board that I will be managing so that I can help assist in any way I can in the completion of these ideas.
11:42 AM
answer to part two: This is probably the best question I have seen so far thank you for bringing this up. I think it is a mixture of both in terms of inactivity and communication. Committee members are volunteers and so expecting a certain level of commitment outside of allotted CSC time can be difficult to promise when applying for these positions. Communication is key in this regard and for smaller committees I think it's important to implement "scrum-like" meetings for them so that all members can be on the same page. For example, media has at most 2 meetings a season(usually before the season starts). Having more meetings during the season can help guide committee members who may be lost or find it hard to communicate their problems back to a point where they feel included in the committee. It can be daunting going into a job you don't know yet so having a support pipeline that isn't just DMing the committee head is crucial for success.
